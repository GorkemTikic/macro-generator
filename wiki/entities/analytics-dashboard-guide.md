# Analytics Dashboard Guide (Leadership PDF)

Leadership-facing PDF document explaining the FD Macro Generator analytics dashboard and the export workbook. Written in first-person voice ("I built this because…") so it can be shared at weekly FD/SL meetings and stand on its own without a verbal walk-through.

## Purpose

- Demonstrate the contribution to upper management without requiring a live demo.
- Justify each design decision in plain English (privacy, free-tier infra, devices-vs-users framing).
- Provide a single artefact to drop into a deck or share over email.

## Document structure

1. **Cover page** — gradient slate-to-blue background, eyebrow text "Internal · FD / SL Tooling", title, sub, and a metadata footer with name + date.
2. **Why I built this** — lede paragraph plus a 2×2 grid of "why-cards": problem statement, leadership view, team view, refused trade-offs (privacy).
3. **The Admin Dashboard — three operational views** — one card per dashboard tab, each with screenshot + caption:
   - Overview ("Is the tool being used, and what for?")
   - Recent Events ("What just happened in the tool?")
   - Devices ("How many real people are actually using this?")
4. **Export Workbook (.xlsx) — five sheets in one click** — overview table + one card per workbook sheet, each with embedded screenshot:
   - Summary · Device Journey · Raw Events · Devices · Ready to Share
5. **How this is built — and why it costs us nothing** — Cloudflare Workers + D1 + GitHub Pages, free-tier, no third-party trackers, no PII.
6. **Closing sign-off** — dark callout box reiterating the leadership takeaway, signed.

## Generation pipeline

The HTML is hand-written and styled with print-specific CSS (`@page { size: A4; margin: 18mm 16mm; }`). The PDF is generated by Microsoft Edge in headless mode — no Node-side dependency, no Chromium install needed (Edge ships on every Windows machine).

```bash
"/c/Program Files (x86)/Microsoft/Edge/Application/msedge.exe" \
  --headless=new \
  --disable-gpu \
  --no-pdf-header-footer \
  --print-to-pdf="docs/analytics-dashboard-guide.pdf" \
  "file:///…/docs/analytics-dashboard-guide.html"
```

Notes:
- `--headless=new` is required on this Edge build; the legacy `--headless` flag silently produced no output.
- The PDF must not be open in another reader (e.g. VS Code's PDF preview) when re-rendering — the file lock is silent and Edge exits 0 without writing.

## Updating the document

1. Edit `docs/analytics-dashboard-guide.html` (text, structure, screenshots).
2. If swapping screenshots, drop the new file at `docs/<NN>-<name>.png` keeping the same filename — the HTML uses relative `<img src="NN-…png" />` references.
3. Re-run the Edge command above (or open the HTML in a browser and "Save as PDF" with default settings).

## Style choices

- A4 portrait, 18 mm vertical / 16 mm horizontal margins.
- Single typography stack: Segoe UI / Helvetica Neue / Arial.
- Brand accent: `#0b5fae` (Cloudflare-adjacent blue), darker `#0f172a` for headings.
- Tab cards have coloured pill labels (slate, amber, green, purple) so each section is visually distinct without relying on chapter numbers.
- All screenshots are full-width with a 1 px border + soft shadow so they feel embedded, not pasted.
